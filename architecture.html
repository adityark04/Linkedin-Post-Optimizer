LinkedIn Post Optimizer

Application Architecture & Data Flow

Core Workflows

Workflow 1: Generate Post (with RAG)

This is the most advanced feature, using Retrieval-Augmented Generation (RAG) to create posts in the user's personal style. It has two phases:

Part A: Building the Knowledge Base (Your Style Library)

User Action: User pastes a successful post into the StyleLibraryManager component and clicks "Add to Library".

Embedding Logic: The addPostToLibrary function is called. It sends the post text to our geminiService.

AI Call: The service calls the Gemini embedding model to convert the text into a numerical vector (an "embedding").

Data Storage: The original text and its new vector embedding are saved together as an object into the browser's Local Storage. This creates a persistent, personal style database for the AI.

Part B: Generating a New Post

User Action: User fills in the 'Goal', 'Tone', and 'Details' form fields in the PostGenerator component and clicks "Generate Posts".

Step 1: Embed Query (Logic): The app takes the 'Goal' and 'Details' and sends them to the Gemini embedding model to create an embedding for the user's *current request*.

Step 2: Retrieve (Logic): The app's cosineSimilarity function compares the user's request embedding to all the post embeddings stored in the Local Storage library.

Step 3: Augment (Logic): The system selects the top 3 most stylistically similar posts from the library. These are the most relevant examples for the current task.

Step 4: Generate (AI Call): The user's request (Goal, Tone, Details) AND the 3 retrieved posts are sent to the powerful gemini-2.5-pro model. The AI is instructed to mimic the style of the provided examples.

Render Result: The AI returns a new post variation, which is displayed to the user in the GeneratedPost component.

Workflow 2: Analyze Post

This workflow provides a direct analysis of a user's draft post.

User Action: User navigates to the 'Revise Post' tab and pastes their draft into the PostAnalyzer component.

AI Call: The draft text is sent directly to the gemini-2.5-pro model with a detailed prompt asking for a structured analysis (score, strengths, improvements, revised version, etc.) in JSON format.

Render Result: The AI returns a JSON object with the full analysis. This data is passed to the PostAnalysisResult component, which formats it into the score gauge, lists, and cards for the user to see.
